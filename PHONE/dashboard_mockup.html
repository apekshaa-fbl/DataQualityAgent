Quality Metrics — Definitions & How They Are Calculated
📊 Coverage
Coverage % = (Records with Phone / Total Records) × 100
Measures how many company–country records have at least one phone — either in PRIMARY_PHONE or OTHER_PHONES. A record counts as covered even if the phone lacks a country code or is a toll-free number. Coverage only measures presence, not quality.
Current: 26.0%  ·  Target: 60%
🎯 Accuracy
Accuracy % = weighted composite score per record, averaged across all records with phone
Accuracy is a weighted composite of 5 signals per phone record:

35% Country code matches region (e.g. +61 for AU)
25% Not a toll-free / premium number (800/888/900)
20% Not a cross-company duplicate
15% Valid E.164 or recognised local format
5% Correct digit count for country

Each record scores 0–100 on these signals combined. Country Accuracy = average across all records with phone in that country.
Current: 58.0%  ·  Target: 80%
✅ Consistency
Consistency % = (Valid-format, Non-premium Phones / Records with Phone) × 100
Measures whether phones are structurally dialable. A phone is consistent if it: (1) passes format validation — has a recognisable structure with correct digit count for its country — and (2) is not a premium or toll-free line (800/888/900 series). Mobile vs landline and missing country code are not penalised here — they are flagged separately as signals.
Current: 63.0%  ·  Target: 75%

Scoring Rules — 0–9 Points
Applied to every phone number independently · Higher = better quality
+3
Country code matches company region
Phone has an E.164 prefix (+XX) and the country code correctly matches the company's REGION_CODE. E.g. +61 for AU, +1 for US/CA, +81 for JP, +65 for SG. The single strongest signal that the number actually belongs to this company.
+2
Not a toll-free or premium number
Phone does not start with a toll-free prefix (800, 888, 877, 866, 855, 844) or premium (900). Toll-free lines are shared service numbers — they rarely identify a specific company's direct contact.
+2
Not a cross-company duplicate
The normalised phone number does not appear in more than one distinct company's PRIMARY_PHONE field. A number shared across multiple FIRMABLE_IDs suggests a data ingestion error or a shared call centre line.
+1
Valid E.164 or recognised local format
Phone passes structural format validation — either it uses a correct E.164 prefix (+XX followed by digits) or it matches a recognised local format for its country (e.g. 0X XXXX XXXX for AU, (0X) XXXX XXXX for JP). Pure strings of ambiguous length score 0 here.
+1
Correct digit count for country
After stripping formatting, the digit count falls within the expected range for the country (e.g. 10 digits for US/CA, 9–10 for AU, 10–11 for JP). Numbers with too few or too many digits score 0 here.
+1
Mobile bonus for micro companies (≤10 emp)
If the company has ≤10 employees globally and the phone is a mobile number, it receives a +1 bonus. Micro businesses frequently operate via mobile — this is the most realistic primary contact for this segment.
Max score: 9 (mobile + micro bonus) · 8 (landline, perfect)
7–9: High quality · 4–6: Medium quality · 0–3: Low quality
